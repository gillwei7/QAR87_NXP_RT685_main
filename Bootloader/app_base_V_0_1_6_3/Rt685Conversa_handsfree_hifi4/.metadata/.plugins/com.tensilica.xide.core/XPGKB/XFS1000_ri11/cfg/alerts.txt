README for Xtensa Tools Patch 640199 Installation
=================================================

This patch is to be applied to Xtensa Tools version 14.0.4 (release RI-2020.4).

This readme has the following sections

- Fixes and enhancements included in this Xtensa Tools patch
- Linux Command Line Installation Procedure
- Windows Command Line Installation Procedure
- Installing Xtensa Tools Patches With Xplorer


Fixes and Enhancements included in this Xtensa Tools patch
----------------------------------------------------------

- sw-debug - xt-gdb for Linux was built without TUI mode enabled. This has now
  been fixed and TUI mode (curses based UI) is now available on Linux

- sw-iss - Bus error responses would not trigger exceptions in the Xtensa ISS
  fast-functional mode. This has been fixed.

- sw-iss - The xtsc_router component in XTSC has been enhanced with an addition
  of the  "wait_on_outstanding_write" parameter to ensure that read requests
  are not issued until all previous writes to the same address have completed.

- sw-iss - Various improvements have been made to the DNA 150 simulation model.

- sw-iss - XTSC libraries compatible with Cadence Xcelium 19.09 have been
  added.

- sw-iss - Programs that use gather/scatter operations on Vision Q7
  configurations with ECC memory integrity option would hang when run in the
  Xtensa Instruction Set Simulator. This has been fixed. 

- sw-iss - When modeling level-sensitive interrupts on Xtensa NX processors,
  the instruction set simulator may incorrectly allow an interrupt to be taken
  again after it has already been cleared at the source. This has been fixed.

- sw-xcc - In certain circumstances when compiling for LX6 or older processors,
  xt-clang could inappropriately combine loads and stores in a FLIX bundle.
  This has been fixed.

- sw-xcc - In certain situations the xt-clang compiler would not honor correct
  ordering of memory access to volatile locations. This has been fixed.

- sw-xcc - The xt-clang compiler was overly restrictive of certain combinations
  of regfile width and ctype alignment in user tie. The restriction has been
  relaxed.

- sw-xcc - The xt-clang compiler may incorrectly remove predicated store
  operations programmed by using ConnX, Fusion G, or Vision DSP intrinsics.
  This has been fixed.


Linux Command Line Installation Procedure
-----------------------------------------

The patch file downloaded from the XPG is a tgz bundle named
XtensaTools_RI_2020_4_patch640199_linux_installer.tgz.

Unpack this tgz file into an empty temp directory and you will
now find an install script called "install-patch" and the actual
patch content which is inside another tgz named
XtensaTools_RI_2020_4_patch640199_linux.tgz

Run the install-patch script in the current directory as follows:

    ./install-patch

The script will prompt you for the path of RI-2020.4 XtensaTools. Enter the path
for the script to complete installation of the patch.


Windows Command Line Installation Procedure
-------------------------------------------

On windows, it is recommended that you use Xplorer to install patches
- see below for more information on how to do this.

The patch file downloaded from the XPG for windows is a simple tgz file
that does not have an installer. If you have appropriate tools that
can unpack tgz files (e.g. a tar that can handle the gzip format)
follow these steps:

- chdir to the directory that contains RI-2020.4 XtensaTools.
  (if you do a dir command at the directory, you should see RI-2020.4-win32)

- untar XtensaTools_RI_2020_4_patch640199_win32.tgz file at the current directory
  to install the patch.


Installing Xtensa Tools Patches With Xplorer
--------------------------------------------

Cadence/Tensilica sometimes releases patches for Xtensa Tools via Xplorer. A
tools patch may be downloaded by Xplorer and offered for installation, or it
may be bundled together with an Xplorer update. Either way, after a tools patch
is downloaded, upon subsequent restart it will be offered for installation 
- if it matches the tools version of the current configuration in the 
Xplorer Active Set.

If you do not accept invitations to download / install  Xplorer or Xtensa Tools 
updates, then you can still use Xplorer to install an Xtensa Tools patch thus:

- Download the appropriate patch file (e.g. from the XPG view) for your platform:

    XtensaTools_RI_2020_4_patch640199_linux.tgz
    XtensaTools_RI_2020_4_patch640199_win32.tgz

    (if you have unpacked the linux *installer.tgz, this is the same file
     as the tgz which is inside that installer)

- From the Xtensa Xplorer Help menu, select Software Updates, and then
  Update Xtensa Tools.

- From the "Which Xtensa Tools" option, navigate to select the target
  version of the Xtensa Tools (the RI-2020.4 release).

- From the "Which Patch" option, navigate to the downloaded patch file.
  Note that if the patch was downloaded using the XPG view to the
  "default" location, then Xplorer should find the latest patch
  automatically.

- Click Update to install the patch.
